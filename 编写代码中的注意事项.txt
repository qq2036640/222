1. 基础语法规范（直接影响测试工具能否解析）
JSON 语法绝对合法：
禁止出现语法错误：缺少逗号、引号未闭合、花括号 / 方括号不匹配、冒号使用错误（必须是英文冒号 :，而非中文 ：）；
特殊字符必须转义：法律条文里的双引号（"）、换行符（\n）、斜杠（/）需用 \ 转义，例如 "原文": "《刑法》第266条：“诈骗公私财物，数额较大的...”" 需改为 "原文": "《刑法》第266条：\"诈骗公私财物，数额较大的...\""；
字段值类型匹配：数值型字段（如刑期、金额）必须是数字（120），禁止用字符串（"120"）；逻辑型字段（如 isPass）必须是 true/false，禁止用 "true"/"false" 或 1/0；
无冗余逗号：数组 / 对象最后一个元素后禁止加逗号（如 ["a","b",] 会直接导致解析失败）。
语言与编码要求：
所有文本字段（法条原文、提示词、错误信息）必须使用简体中文，禁止混用英文 / 繁体；
JSON 文件编码统一为 UTF-8，禁止使用 GBK/GB2312（避免中文乱码）；
字段名称、规则名称、节点名称统一使用中文 + 下划线（如 基础_案件地区代码），禁止使用特殊字符（@、#、$）或空格。
2. 数据类型禁忌（比赛明确禁止的错误类型）
禁止出现 unhashable type: 'list' 错误：
核心要求：所有地区标准、数值参数、规则条件均需用独立键值对 / 节点存储，禁止用数组（list）嵌套存储，例如：
❌ 错误示例："地区标准": ["北京": {"诈骗较大":5000}, "浙江": {"诈骗较大":6000}]
✅ 正确示例："基础_地区标准_北京_诈骗较大": 5000, "基础_地区标准_浙江_诈骗较大": 6000
数组仅用于「无关联的清单类数据」（如罪名列表 ["诈骗罪","盗窃罪"]），禁止用于需要匹配 / 计算的核心数值。
禁止类型混用：
同一字段在不同规则 / 节点中类型必须统一，例如 基础_案发时间 若定义为字符串（"2024-01-01"），则所有引用该字段的规则中均需按字符串处理，禁止部分规则按数值、部分按字符串。
3. 逻辑设计禁忌（比赛审计核心检查点）
绝对禁止逻辑自环：
规则 / 节点之间必须是单向推导链，禁止循环调用（如规则 A 依赖规则 B 的结果，规则 B 又依赖规则 A 的结果）；
检查方法：梳理所有规则的「输入→输出」链路，确保链路是线性 / 分支型（如前置验证→地区匹配→罪名认定→量刑计算），无闭环。
禁止核心逻辑依赖代码：
即使不要求低代码部署，比赛仍禁止将核心法律规则（如量刑计算、罪名竞合判断）封装为纯代码逻辑嵌入 JSON；
所有核心逻辑必须通过「规则字段」（条件判断、算术运算、逻辑运算）显性表达，代码仅能用于辅助（如数据清洗、格式转换），且需在 JSON 中注明代码用途（无隐藏逻辑）。
规则层级与复杂度控制：
单条规则的条件数≤8 个，规则嵌套层级≤5 层（避免测试工具解析超时）；
禁止「一条规则覆盖所有逻辑」（如用 1 条规则完成量刑计算 + 缓刑判断），需按功能拆分（基准刑计算、调节系数、缓刑审查各为独立规则）。
4. 比赛合规要求（白盒审计 / 反混淆）
无模糊 / 混淆表述：
规则名称、节点名称、法律依据必须清晰可追溯，禁止用「规则 1、节点 A」等模糊命名，需按「规则_功能模块_具体场景」命名（如 规则_缓刑四要件审查）；
所有法律依据必须标注「法条名称 + 条款 + 原文」（如 "法律依据": "《刑法》第72条：对于被判处拘役、三年以下有期徒刑的犯罪分子，同时符合下列条件的，可以宣告缓刑..."），禁止仅标注「刑法第 72 条」。
Prompt 可见性（若涉及 LLM 节点）：
若 JSON 中包含 LLM 节点，提示词（Prompt）必须直接嵌入节点字段（如 "prompt": "根据案件事实判断基础_主体适格是否成立"），禁止通过外部 API / 链接引用，确保白盒审计可见。
无违规字段 / 内容：
禁止包含诱导、攻击、歧视性内容，禁止伪造 / 篡改法条原文；
禁止包含比赛未要求的字段（如开发者信息、测试数据），仅保留「案由、法律法规、规则、节点」四大核心模块（链图 JSON）和「任务名称、适用声明、核心任务节点清单、输出结果规范」（任务节点 JSON）。
5. 法律维度准确性（影响法条还原度评分）
法条原文精准无删减：
引用的刑法条款、司法解释、地区标准必须与官方原文一致，禁止修改措辞（如「从旧兼从轻」不能写成「从新兼从轻」）；
溯及力判断、数罪并罚规则、缓刑条件必须严格对应法条，禁止主观修改法律逻辑（如缓刑考验期计算需严格按《刑法》第 73 条，禁止自定义公式）。
数值标准与地区匹配准确：
30 省诈骗 / 盗窃数额标准需与各省高院 2022 年后发布的有效文件一致，禁止混用旧标准（如浙江诈骗 “数额较大” 为 6000 元，禁止写成 5000 元）；
刑期、系数等数值需符合量刑指导意见（如从犯调节系数为 - 0.2~-0.5，禁止超出该范围）。
